Objective

Develop foundational Linux skills used by Security Operations Center (SOC) analysts by practicing file system navigation, file management, and basic investigation workflows within an Ubuntu virtual machine.

Environment
Ubuntu Desktop LTS
Oracle VirtualBox
Windows 11 Host
Skills Practiced
Navigating the Linux file system
Creating and managing directories
Creating, viewing, copying, moving, and deleting files
Searching for files
Organizing an investigation workspace
Documenting technical tasks
Linux Commands Used
Command	Purpose
pwd	Display the current working directory
ls	List files and folders
ls -la	Display all files, including hidden files
cd	Change directories
mkdir	Create directories
touch	Create files
nano	Edit text files
cat	View file contents
cp	Copy files
mv	Move files
rm	Remove files
rmdir	Remove empty directories
find	Locate files by name
Lab Activities

During this mission, I created a structured investigation workspace using Linux commands. I practiced navigating directories, organizing evidence, creating and managing files, and locating files using the find command. Throughout the lab, I focused on using Linux commands in scenarios that reflect common SOC analyst tasks, such as preserving evidence, organizing investigation data, and verifying file locations before performing actions.

SOC Analyst Takeaways

This project strengthened my understanding of Linux fundamentals used in Security Operations Centers. I learned how to navigate the Linux file system efficiently, organize investigation data, preserve original evidence by working on copies, and quickly locate files during an investigation. These foundational skills will support future projects involving log analysis, threat hunting, and incident response.
